# Codex maintenance metrics

This log is intentionally empty until the opt-in maintenance workflow has been run on a real public Issue and the maintainer has reviewed the result. Do not backfill estimates as measurements.

## Measurement definitions

- **Input/output tokens:** values returned by the Responses API `usage` object.
- **Accepted:** the maintainer used the proposal to create or materially revise a regression fixture or test.
- **Partially accepted:** useful classification or test outline, but substantial manual correction was required.
- **Rejected:** no proposal content was used.
- **Time saved:** manual baseline minutes minus actual reviewed-workflow minutes, both recorded by the same maintainer for comparable work.
